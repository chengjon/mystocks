# Claude协作框架说明

## 概述
本目录用于两个Claude实例之间的协作，通过文件交换的方式共享想法、代码和文档。

## 目录结构说明

### ideas/ - 设计思路和想法交流
- 用于记录和交换设计思路
- 存放头脑风暴结果
- 记录创新想法和解决方案

### code/ - 代码共享和审查
- 共享代码片段和完整实现
- 代码审查意见和建议
- 最佳实践和模式分享

### docs/ - 文档和规范
- 技术文档和规范
- 设计文档和架构图
- 使用说明和教程

### tasks/ - 任务分配和进度跟踪
- 任务分配和状态跟踪
- 项目计划和里程碑
- 问题记录和解决方案

## 协作流程

### 1. 任务分配
1. 在tasks目录创建任务描述文件
2. 明确任务负责人和截止时间
3. 定期更新任务状态

### 2. 想法交流
1. 在ideas目录创建想法文档
2. 使用Markdown格式详细描述
3. 添加标签和分类便于检索

### 3. 代码共享
1. 在code目录放置代码文件
2. 添加详细注释和使用说明
3. 进行代码审查并记录意见

### 4. 文档协作
1. 在docs目录创建和更新文档
2. 使用版本控制跟踪变更
3. 确保文档与代码同步更新

## 文件命名规范
- 使用有意义的英文文件名
- 添加日期前缀：YYYYMMDD_filename.extension
- 使用小写字母和下划线分隔

## 通信协议
- 使用JSON格式交换结构化数据
- 使用Markdown格式交换文档和想法
- 使用标准注释格式进行代码交流
