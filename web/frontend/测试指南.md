## 环境准备

### 1. 系统要求

- **操作系统**: Linux / macOS / Windows (WSL2)
- **Node.js**: 16.x 或更高版本
- **npm**: 8.x 或更高版本
- **PM2**: 5.x 或更高版本
- **浏览器**: Chromium 90+, Firefox 88+, Safari 14+（用于 Playwright）

### 2. 安装依赖

```bash
# 进入前端项目目录
cd /opt/claude/mystocks_spec/web/frontend

# 安装项目依赖（包含 Playwright）
npm install

# 安装 PM2（全局）
npm install -g pm2

# 安装 serve（用于生产构建服务）
npm install -g serve

# 验证安装
node --version   # >= 16.0.0
npm --version    # >= 8.0.0
pm2 --version    # >= 5.0.0
```

### 3. 首次运行 Playwright

```bash
# 安装 Playwright 浏览器驱动
npx playwright install

# 验证浏览器安装
npx playwright --version
```

## 部署步骤

### 方式一：使用自动化脚本（推荐）

```bash
# 1. 构建生产版本
npm run build

# 2. 启动 PM2 服务
./start.sh

# 3. 查看服务状态
pm2 list

# 4. 查看日志
pm2 logs mystocks-frontend --lines 50
```

### 方式二：手动部署

```bash
# 1. 构建项目
npm run build

# 2. 创建日志目录
mkdir -p logs

# 3. 使用 serve 启动生产构建（PM2 管理）
pm2 start ecosystem.config.js --env production

# 4. 保存 PM2 进程列表
pm2 save

# 5. 验证服务运行
curl http://localhost:8080
```

### 3. 验证部署成功

```bash
# 检查 PM2 进程状态（应为 online）
npm run pm2:status

# 检查端口监听（应显示 PM2 进程 PID）
lsof -i :8080

# 检查服务响应
curl -I http://localhost:8080

# 应返回：
# HTTP/1.1 200 OK
# Content-Type: text/html
```

## 失败排查

### 场景 1: PM2 启动失败

**症状**: 运行 `./start.sh` 后应用无法启动

**排查步骤**:

```bash
# 1. 检查 Node.js 版本
node --version  # 应 >= 16.x

# 2. 检查构建产物
ls -la dist/
# 应包含 index.html 和 assets 目录

# 3. 检查端口占用
lsof -i :8080
# 如果被占用，找出占用进程并终止

# 4. 查看详细错误日志
pm2 logs mystocks-frontend --lines 100 --err

# 5. 尝试手动启动（绕过 PM2）
cd dist && python3 -m http.server 8080
# 访问 http://localhost:8080 验证
```

**解决方案**:

```bash
# 端口被占用时，终止占用进程
PID=$(lsof -t -i:8080)
kill -9 $PID

# 重新启动
./stop.sh && ./start.sh
```

### 场景 2: SCSS 编译错误

**症状**: 页面样式未加载，日志中有 `sass-loader` 错误

**排查步骤**:

```bash
# 1. 检查错误日志
pm2 logs mystocks-frontend --err | grep -i scss

# 2. 检查 theme-tokens.scss 文件
ls -la src/styles/theme-tokens.scss

# 3. 验证 SCSS 变量语法
cat src/styles/theme-tokens.scss | head -20

# 4. 检查是否缺少依赖
npm list sass
```

**解决方案**:

```bash
# 重新安装依赖
rm -rf node_modules package-lock.json
npm install

# 重新构建
npm run build

# 重启 PM2
./stop.sh && ./start.sh
```

### 场景 3: Design Token 未加载

**症状**: 测试失败，提示 CSS 变量未定义

**排查步骤**:

```bash
# 1. 检查 theme-tokens.scss 是否存在
cat src/styles/theme-tokens.scss | grep "color-accent"

# 2. 检查组件是否正确导入
grep -r "@import.*theme-tokens" src/

# 3. 在浏览器中检查 CSS 变量
# 打开 http://localhost:8080
# 在开发者工具 Console 中运行：
getComputedStyle(document.body).getPropertyValue('--color-accent')
# 应返回: #D4AF37 或 rgb(212, 175, 55)
```

**解决方案**:

```bash
# 检查组件样式导入
grep -A2 "<style" src/views/Dashboard.vue | grep "@import"

# 如果缺失，添加：
# @import '@/styles/theme-tokens.scss';

# 重新构建和部署
npm run build && ./stop.sh && ./start.sh
```

### 场景 4: 视觉对比测试失败

**症状**: 截图对比差异超过阈值

**排查步骤**:

```bash
# 1. 查看失败截图
ls -la test-results/

# 2. 检查 Playwright 截图目录
ls -la tests/screenshots/

# 3. 更新基准截图（如果样式变更合理）
npx playwright test --update-snapshots

# 4. 检查浏览器缩放
# 确保浏览器缩放为 100%
```

**解决方案**:

```bash
# 如果样式变更符合预期，更新基准截图
npx playwright test --update-snapshots

# 如果是真实问题，检查样式
npm run build && ./stop.sh && ./start.sh
```

### 场景 5: ECharts 渲染失败

**症状**: ChartContainer 测试失败，图表无法显示

**排查步骤**:

```bash
# 1. 检查 ECharts 是否加载
curl http://localhost:8080 | grep "echarts"

# 2. 在浏览器 Console 中检查
# 打开 http://localhost:8080/market
# Console: window.echarts
# 应返回 ECharts 对象，而非 undefined

# 3. 检查图表配置
# Console: document.querySelector('.chart-container')
# 应找到图表容器元素
```

**解决方案**:

```bash
# 检查 ECharts 依赖
npm list echarts

# 如果缺失，重新安装
npm install echarts@^5.5.0

# 重新构建
npm run build
```

## 常见问题

### Q1: 测试时提示 "Cannot find module '@/styles/theme-tokens.scss'"

**原因**: TypeScript 路径别名未正确配置

**解决方案**:

```bash
# 检查 vite.config.js 中的别名配置
grep -A5 "resolve:" vite.config.js | grep "@"

# 确保有配置：
# resolve: {
#   alias: {
#     '@': path.resolve(__dirname, 'src')
#   }
# }
```

### Q2: Playwright 无法连接到 localhost:8080

**原因**: PM2 服务未启动或端口错误

**解决方案**:

```bash
# 1. 检查 PM2 服务
pm2 list

# 2. 如果未启动，启动服务
./start.sh

# 3. 如果端口错误，修改配置
vim ecosystem.config.js
# 修改 port: 8080 为其他端口

# 4. 更新 Playwright 配置中的 BASE_URL
export BASE_URL=http://localhost:3000
npm run test:phase3
```

### Q3: 视觉对比测试频繁失败

**原因**: 浏览器渲染差异、字体加载问题

**解决方案**:

```bash
# 1. 使用单一浏览器测试
npm run test:e2e:chromium

# 2. 调整截图阈值
# 编辑 playwright.config.ts:
# threshold: 0.01,  # 从 0.001 调整为 0.01 (1%)

# 3. 等待字体加载
# 在测试中添加:
# await page.waitForLoadState('networkidle');
# await page.waitForTimeout(1000);
```

### Q4: 测试超时

**原因**: 页面加载慢、API 请求阻塞

**解决方案**:

```bash
# 1. 增加测试超时时间
# 编辑 playwright.config.ts:
# timeout: 60 * 1000,  # 从 30s 增加到 60s

# 2. 使用 mock 数据
npm run dev:mock

# 3. 跳过网络等待
# 在测试中使用:
# await page.goto('/', { waitUntil: 'domcontentloaded' });
```

### Q5: PM2 日志文件过大

**原因**: 日志未轮转，长期积累

**解决方案**:

```bash
# 1. 清空旧日志
> /opt/claude/mystocks_spec/var/log/pm2-out.log
> /opt/claude/mystocks_spec/var/log/pm2-error.log

# 2. 配置 PM2 日志轮转
# 编辑 ecosystem.config.js:
# log_date_format: 'YYYY-MM-DD',
# error_file: '/opt/claude/mystocks_spec/var/log/pm2-error-<DATE>.log',

# 3. 定期清理日志（cron）
# 0 0 * * * find /opt/claude/mystocks_spec/var/log -name "pm2-*.log" -mtime +7 -delete
```
