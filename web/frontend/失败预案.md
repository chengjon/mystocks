## 样式偏差失败

### 症状描述

Playwright 视觉对比测试失败，提示截图像素差异超过阈值（0.1%）

### 可能原因

1. **浏览器渲染差异**: 不同浏览器对 CSS 的渲染略有不同
2. **字体加载问题**: Inter 字体未加载完成，使用了系统字体
3. **动画过渡**: 页面元素有动画未完成，导致截图不一致
4. **时间相关元素**: 页面包含实时时间、动态数据等

### 排查步骤

```bash
# 1. 查看失败截图对比
open playwright-report/index.html
# 找到失败的测试用例，查看 "Diff" 图像

# 2. 检查差异类型
# - 颜色差异：样式问题
# - 布局差异：渲染问题
# - 文本差异：国际化或数据问题

# 3. 在浏览器中手动验证
# 打开 http://localhost:8080
# 检查页面外观是否符合预期

# 4. 检查浏览器控制台
# 打开 DevTools > Console
# 查找 CSS 加载错误、字体加载错误
```

### 解决方案

#### 方案 A: 更新基准截图（如果样式变更符合预期）

```bash
# 确认样式变更符合 ArtDeco 设计规范后
npx playwright test --update-snapshots

# 验证更新后的基准截图
npm run test:artdeco-style
```

#### 方案 B: 调整截图阈值

```bash
# 编辑 playwright.config.ts
export default defineConfig({
  expect: {
    // 从 0.001 (0.1%) 调整为 0.01 (1%)
    threshold: 0.01,
  },
});
```

#### 方案 C: 等待页面稳定后再截图

```typescript
// 在测试用例中添加等待
test('should match ArtDeco style', async ({ page }) => {
  await page.goto('/');

  // 等待字体加载
  await page.waitForLoadState('networkidle');
  await page.waitForTimeout(1000);

  // 等待特定元素渲染
  await page.waitForSelector('.page-title');
  await page.waitForSelector('.main-content');

  // 然后再截图
  await expect(page).toHaveScreenshot();
});
```

### 预防措施

1. **统一测试环境**: 使用相同版本的浏览器和操作系统
2. **禁用动画**: 测试时禁用 CSS 动画和过渡
3. **Mock 数据**: 使用固定数据，避免时间变化
4. **CI 环境**: 在 CI 中使用无头浏览器（headless: true）避免 GUI 差异

## 进程崩溃失败

### 症状描述

PM2 进程状态为 `stopped` 或 `errored`，应用无法访问

### 可能原因

1. **构建错误**: SCSS 编译失败、TypeScript 类型错误
2. **依赖缺失**: node_modules 不完整
3. **端口占用**: 8080 端口被其他进程占用
4. **内存溢出**: 应用内存使用超过限制（1GB）
5. **未捕获异常**: 运行时错误导致进程崩溃

### 排查步骤

```bash
# 1. 检查 PM2 进程状态
pm2 list
pm2 show mystocks-frontend

# 2. 查看错误日志
pm2 logs mystocks-frontend --err --lines 100
cat /opt/claude/mystocks_spec/var/log/pm2-error.log | tail -50

# 3. 检查端口占用
lsof -i :8080

# 4. 尝试手动启动（不用 PM2）
cd dist && python3 -m http.server 8081
# 访问 http://localhost:8081 验证构建产物是否正常

# 5. 检查内存使用
pm2 monit
```

### 解决方案

#### 方案 A: 重新构建项目

```bash
# 清理并重新安装依赖
rm -rf node_modules dist package-lock.json
npm install

# 重新构建
npm run build

# 重启 PM2
./stop.sh && ./start.sh
```

#### 方案 B: 修复端口占用

```bash
# 找到占用端口的进程
PID=$(lsof -t -i:8080)
ps -p $PID

# 终止占用进程
kill -9 $PID

# 重启 PM2
./start.sh
```

#### 方案 C: 增加内存限制

```javascript
// 编辑 ecosystem.config.js
module.exports = {
  apps: [{
    name: 'mystocks-frontend',
    // 增加内存限制从 1G 到 2G
    max_memory_restart: '2G',
    // 或完全移除内存限制
    // max_memory_restart: undefined,
  }]
};
```

#### 方案 D: 修复运行时错误

```bash
# 查看具体错误信息
pm2 logs mystocks-frontend --lines 200

# 常见错误和修复：

# 1. CSS 导入错误
# Error: Failed to resolve importer
# 修复: 确保 theme-tokens.scss 路径正确

# 2. ECharts 未定义
# Error: echarts is not defined
# 修复: npm install echarts@^5.5.0

# 3. 路由错误
# Error: Failed to load route
# 修复: 检查 router/index.js 配置
```

### 预防措施

1. **健康检查**: 配置 PM2 健康检查，自动重启崩溃进程
2. **日志监控**: 定期检查错误日志，及时发现问题
3. **资源限制**: 合理设置内存和 CPU 限制
4. **错误处理**: 应用中添加全局错误处理

```javascript
// ecosystem.config.js 中配置
module.exports = {
  apps: [{
    name: 'mystocks-frontend',
    autorestart: true,
    max_restarts: 10,
    min_uptime: '10s',
    restart_delay: 4000,
    kill_timeout: 5000,
    watch: false,
  }]
};
```

## ECharts 渲染失败

### 症状描述

图表无法显示，或显示为空白容器，控制台有 ECharts 错误

### 可能原因

1. **ECharts 未加载**: CDN 或构建时未正确包含
2. **容器尺寸问题**: 图表容器高度或宽度为 0
3. **配置错误**: ECharts 配置有语法错误
4. **数据问题**: 图表数据格式不正确
5. **主题未应用**: ArtDeco 主题配置未生效

### 排查步骤

```bash
# 1. 检查 ECharts 是否加载
curl http://localhost:8080 | grep "echarts"

# 或在浏览器控制台中检查
window.echarts
# 应返回 ECharts 对象，而非 undefined

# 2. 检查图表容器
# 打开 http://localhost:8080/market
# Console:
document.querySelector('.chart-container')
# 应找到元素

# 3. 检查容器尺寸
# Console:
const container = document.querySelector('.chart-container');
getComputedStyle(container).width;
getComputedStyle(container).height;
# 应有非零值

# 4. 查看控制台错误
# 打开 DevTools > Console
# 查找 ECharts 相关错误
```

### 解决方案

#### 方案 A: 重新安装 ECharts

```bash
# 卸载旧版本
npm uninstall echarts

# 安装指定版本
npm install echarts@^5.5.0

# 重新构建
npm run build

# 重启服务
./stop.sh && ./start.sh
```

#### 方案 B: 修复容器尺寸

```vue
<!-- ChartContainer.vue -->
<template>
  <div ref="chartContainerRef" class="chart-container" :style="{ height: height }">
    <!-- 确保有明确高度 -->
  </div>
</template>

<style scoped lang="scss">
.chart-container {
  min-height: 400px;  // 添加最小高度
  width: 100%;
}
</style>
```

#### 方案 C: 修复 ECharts 配置

```typescript
// ChartContainer.vue 中修复配置
const getChartOption = (): EChartsOption => {
  const baseOption: EChartsOption = {
    backgroundColor: 'transparent',  // 透明背景

      trigger: props.chartType === 'pie' ? 'item' : 'axis',
      backgroundColor: 'rgba(26, 26, 26, 0.95)',  // ArtDeco 深色背景
      borderColor: '#D4AF37',  // ArtDeco 金色边框
      textStyle: {
        color: '#E5E5E5'  // 高对比度文字
      }
    },
    // ... 其他配置
  };

  return baseOption;
};
```

#### 方案 D: 延迟初始化图表

```typescript
// 确保容器已渲染后再初始化
onMounted(async () => {
  if (!props.lazy) {
    await nextTick();  // 等待 DOM 更新
    await page.waitForTimeout(100);  // 额外等待确保容器有尺寸
    initChart();
  }

```

### 预防措施

1. **版本锁定**: 在 package.json 中锁定 ECharts 版本
2. **尺寸检查**: 初始化前检查容器尺寸
3. **错误处理**: 添加 try-catch 捕获初始化错误
4. **降级方案**: 提供静态图表作为降级选项

## Design Token 未加载

### 症状描述

页面样式为默认值，没有应用 ArtDeco 金色主题，测试提示 CSS 变量未定义

### 可能原因

1. **theme-tokens.scss 未导入**: 组件中缺少 @import 语句
2. **路径错误**: @import 路径不正确
3. **构建配置错误**: Vite 未正确处理 SCSS 文件
4. **CSS 加载顺序**: theme-tokens.scss 在其他样式之后加载

### 排查步骤

```bash
# 1. 检查 theme-tokens.scss 文件
ls -la src/styles/theme-tokens.scss

# 2. 验证文件内容
head -20 src/styles/theme-tokens.scss
# 应包含: --color-accent: #D4AF37;

# 3. 检查组件导入
grep -r "@import.*theme-tokens" src/views/
grep -r "@import.*theme-tokens" src/components/

# 4. 在浏览器中检查 CSS 变量
# 打开 http://localhost:8080
# Console:
getComputedStyle(document.documentElement).getPropertyValue('--color-accent')
# 应返回: #D4AF37
```

### 解决方案

#### 方案 A: 添加缺失的 @import

```vue
<!-- 在每个组件的 <style> 标签顶部添加 -->
<style scoped lang="scss">
// Phase 3.3: Design Token Migration
@import '@/styles/theme-tokens.scss';

// 原有样式...
</style>
```

#### 方案 B: 修复路径别名

```javascript
// vite.config.js
export default defineConfig({
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src'),
      // 确保 @ 别名正确配置
    },
  },
});
```

#### 方案 C: 检查构建产物

```bash
# 1. 清理并重新构建
rm -rf dist
npm run build

# 2. 检查构建产物中的 CSS
grep -r "color-accent" dist/assets/*.css

# 3. 检查 CSS 是否包含 Design Token
grep "color-accent" dist/assets/index-*.css | head -5
```

### 预防措施

1. **ESLint 规则**: 添加规则检查所有 Vue 文件包含 @import
2. **构建检查**: 在构建脚本中验证 Design Token 存在
3. **自动化测试**: 使用 Playwright 自动检查 CSS 变量

## 移动端代码残留

### 症状描述

测试失败，提示发现移动端 @media 查询或 el-col-xs 组件

### 可能原因

1. **代码未清理**: 旧的响应式代码未删除
2. **Element Plus 默认样式**: 组件库自带的响应式类
3. **全局样式污染**: 全局 CSS 包含移动端媒体查询
4. **组件继承**: 组件继承了移动端样式

### 排查步骤

```bash
# 1. 搜索移动端媒体查询
grep -r "@media.*max-width.*768" src/
grep -r "@media.*max-width.*375" src/

# 2. 搜索 Element Plus 移动端组件
grep -r "el-col-xs" src/
grep -r "el-col-sm" src/

# 3. 检查全局样式
grep -r "@media" src/styles/

# 4. 使用 Playwright 检查
npm run test:mobile-cleanup
```

### 解决方案

#### 方案 A: 删除移动端媒体查询

```scss
// 删除类似的代码
@media (max-width: 768px) {
  .container {
    padding: 8px;  // 删除这些
  }
}

// 替换为桌面专用注释
// ============================================
//   DESIGN NOTE - 设计说明
//   本项目仅支持桌面端，不包含移动端响应式代码
// ============================================
```

#### 方案 B: 移除 Element Plus 移动端类

```vue
<!-- 替换 -->
<el-col :xs="24" :sm="12" :md="8">
  <!-- 内容 -->
</el-col>

<!-- 为 -->
<el-col :span="8">
  <!-- 内容 -->
</el-col>

<!-- 或完全不使用 el-col，使用自定义布局 -->
<div class="grid-item">
  <!-- 内容 -->
</div>
```

#### 方案 C: 配置 Element Plus 禁用响应式

```javascript
// main.js 或 app.js
import ElementPlus from 'element-plus'

app.use(ElementPlus, {
  // 禁用响应式类名前缀（如果 Element Plus 支持）
  size: 'large',  // 统一使用大尺寸（桌面）
  zIndex: 3000,
})
```

### 预防措施

1. **Pre-commit Hook**: 添加 Git Hook 阻止提交移动端代码
2. **ESLint 规则**: 添加规则检测 @media 查询
3. **代码审查**: 人工审查所有新增代码

## 中国股市颜色错误

### 症状描述

上涨显示为绿色，下跌显示为红色（西方风格），或颜色未正确应用

### 可能原因

1. **颜色变量错误**: --color-stock-down 和 --color-stock-up 定义错误
2. **条件判断错误**: 正负判断逻辑错误
3. **CSS 类名错误**: positive/negative 类应用错误
4. **第三方组件**: 第三方图表库使用默认颜色

### 排查步骤

```bash
# 1. 检查 Design Token 定义
grep "color-stock" src/styles/theme-tokens.scss

# 2. 在浏览器中验证
# Console:
getComputedStyle(document.documentElement).getPropertyValue('--color-stock-down')
# 应返回: #EB4436 (红)

getComputedStyle(document.documentElement).getPropertyValue('--color-stock-up')
# 应返回: #10B981 (绿)

# 3. 检查实际应用
# 打开 http://localhost:8080/stocks
# 找到显示涨幅的元素
# 检查其 computed color 是否为红色
```

### 解决方案

#### 方案 A: 修正 Design Token 定义

```scss
// src/styles/theme-tokens.scss
:root {
  // 中国股市颜色: 红涨绿跌
  --color-stock-down: #EB4436;  // 红色 (上涨)
  --color-stock-up: #10B981;    // 绿色 (下跌)

  // 确保不是西方风格:
  // --color-stock-up: #EB4436;  // 错误! (这是西方风格)
  // --color-stock-down: #10B981; // 错误!
}
```

#### 方案 B: 修正条件判断逻辑

```vue
<!-- Stocks.vue 或类似组件 -->
<template>
  <span :class="getChangeClass(row.change_pct)">
    {{ row.change_pct }}%
  </span>
</template>

<script setup lang="ts">
const getChangeClass = (value: number | string | null | undefined) => {
  if (value === null || value === undefined || value === '--') return ''

  const numValue = typeof value === 'string' ? parseFloat(value) : value

  // 正确的中国股市判断: 红涨绿跌
  return numValue > 0 ? 'positive' : numValue < 0 ? 'negative' : ''
}
</script>

<style scoped lang="scss">
.positive {
  color: var(--color-stock-down);  // 红色 (上涨)
}

.negative {
  color: var(--color-stock-up);    // 绿色 (下跌)
}
</style>
```

#### 方案 C: ECharts 配置修正

```typescript
// ECharts 中国股市颜色配置
const chartOption = {
  series: [{
    type: 'line',
    itemStyle: {
      color: (params) => {
        // 上涨为红色
        return params.value >= 0 ? '#EB4436' : '#10B981'
      }
    },
    areaStyle: {
      color: (params) => {
        // 面积图颜色
        return params.value >= 0 ? 'rgba(235, 68, 54, 0.3)' : 'rgba(16, 185, 129, 0.3)'
      }
    }
  }]
}
```

### 预防措施

1. **视觉测试**: 添加截图测试验证股票颜色
2. **数据测试**: 使用测试数据验证正负显示
3. **文档说明**: 在代码注释中说明"红涨绿跌"
