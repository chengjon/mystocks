## 🚀 快速导航

### 架构师/技术负责人

3. 指导团队遵循 **[前端设计与组件规范.md](./前端设计与组件规范.md)**

### 前端开发人员
- 重点学习 **[前端设计与组件规范.md](./前端设计与组件规范.md)**

- 参与架构设计评审，提供数据库专业建议

## 📝 文档维护规范

### 更新频率
- **整体架构设计.md**: 架构重大变更时更新
- **前端设计与组件规范.md**: 前端规范变更时更新
- **数据库架构与运维指南.md**: 数据库架构变更时更新
- **架构迭代优化报告.md**: 每个版本发布后更新

### 责任人
- **架构师**: 整体架构设计、架构迭代报告
- **前端负责人**: 前端设计与组件规范
- **数据库负责人**: 数据库架构与运维指南
- **各模块负责人**: 对应模块的设计文档

### 质量要求
1. **架构图清晰**: 使用标准架构图，保持图表更新
2. **技术细节准确**: 确保技术选型、设计模式描述准确
3. **设计决策说明**: 重要架构决策要有充分的理由说明
4. **版本兼容性**: 记录架构变更对现有系统的影响
