# ArtDeco Standardization P1 Tasks

> **说明**
> 本文是 [`ARTDECO_STANDARDIZATION_PLAN.md`](/opt/claude/mystocks_spec/docs/plans/ARTDECO_STANDARDIZATION_PLAN.md) 的 P1 可执行任务清单。
> P1 的定位是“样板组件批次治理”，前提是 P0 已经完成：
> token 缺口已识别、Base / Domain 边界已冻结、首批高频组件样板已确定。

---

## 1. P1 目标

P1 不是全站收敛，而是通过首批高频组件建立可复制样板。

P1 需要达成的结果：

1. 把 P0 识别出的关键 token 缺口补到位
2. 让首批高频组件符合 Base / Domain 分层规则
3. 为首批样板组件补最小视觉与交互回归保护
4. 为 P2 的 CLI 固化提供稳定模板

---

## 2. P1 工作原则

- 只治理首批样板组件，不扩散成全站大修
- 视觉收敛和测试加固同步推进
- 组件结构调整优先于 cosmetic 微调
- 新增 token 必须服务于真实复用场景
- 不在 P1 阶段提前开发 CLI

---

## 3. P1 工作包

### 工作包 A：补齐首批样板组件所需 token

目标：

- 让首批样板组件不再依赖明显重复的字面量视觉值

任务：

- 根据 P0 token 缺口清单补齐首批样板组件所需 token
- 统一高频颜色、边框、阴影、间距、状态语义
- 合并重复 token 命名，淘汰低价值别名

产物：

- 更新后的 `artdeco-tokens.scss`
- token 使用与迁移说明

验收：

- 样板组件中的高频硬编码视觉值显著下降
- 新增 token 能被多个组件复用

### 工作包 B：完成首批样板组件结构收敛

目标：

- 让首批样板组件成为 Base / Domain 分层规则的真实样板

任务：

- 对首批样板组件逐个检查其职责边界
- 将误落在 Base 层的业务逻辑迁出
- 对不清晰的组件命名或目录归属进行收敛
- 明确样板组件的输入输出和组合关系

产物：

- 首批样板组件的结构调整结果
- 组件边界说明

验收：

- 样板组件能被明确归入 Base 或 Domain
- Base 层中不再残留明显业务逻辑
- Domain 层不再依赖隐式样式假设

### 工作包 C：为样板组件补最小测试保护

目标：

- 让首批样板组件具备最小回归防护，避免视觉收敛后再次漂移

任务：

- 为关键组件补交互测试
- 为核心视觉组件补最小视觉快照或回归保护
- 将样式门禁、组件测试与现有测试体系挂接

建议优先覆盖：

- 高交互组件
- 高复用容器组件
- 最容易因为样式漂移影响全局的组件

产物：

- 样板组件测试文件
- 对应门禁或测试入口

验收：

- 样板组件具备最小可执行回归保护
- 结构改动和样式改动能被测试捕捉

### 工作包 D：形成可复制样板包

目标：

- 不只完成一批组件治理，还要让这些组件能作为后续平台化模板

任务：

- 归档样板组件的目录结构、样式组织、测试组织方式
- 形成后续组件治理参考
- 为 P2 CLI 需要固化的模板字段做初步梳理

产物：

- 样板组件模板说明
- P2 CLI 输入项候选清单

验收：

- 团队可以基于样板组件复制新的规范化实现
- P2 不需要重新反推组件模板结构

---

## 4. P1 建议批次

### 批次 1：基础高频组件

建议优先类型：

- Button
- Card
- Input
- Badge / Status

目标：

- 优先稳住全站最常见的基础表达

### 批次 2：骨架与通用容器

建议优先类型：

- Breadcrumb
- Loading Overlay
- 通用布局容器

目标：

- 稳住页面骨架层的一致性

### 批次 3：高频业务域组合组件

建议优先类型：

- 交易管理样板组件
- 策略管理样板组件
- 系统状态样板组件

目标：

- 验证 Domain 层组件在真实业务里的治理方式

---

## 5. P1 不做什么

P1 阶段明确不做以下事情：

- 不试图一次性治理所有 ArtDeco 组件
- 不把视觉收敛扩展为大范围页面重画
- 不先写 CLI 再补样板组件
- 不用临时 patch 代替结构收敛
- 不用测试豁免掩盖明显结构问题

---

## 6. P1 完成标准

当以下条件同时成立时，可判定 P1 完成：

- 首批样板组件所需 token 已补齐
- 首批样板组件已符合 Base / Domain 边界
- 首批样板组件具备最小测试保护
- 已形成足够稳定的样板，可供 P2 脚手架固化

---

## 7. 一句话总结

P1 的核心不是“做更多组件”，而是先把第一批真正高频、真正可复用的组件治理成平台样板。
