# ArtDeco Token Gap Audit Template

> **说明**
> 本模板用于执行 [`ARTDECO_STANDARDIZATION_P0_TASKS.md`](/opt/claude/mystocks_spec/docs/plans/ARTDECO_STANDARDIZATION_P0_TASKS.md) 中的 token 缺口盘点工作。
> 目标不是直接修改 token，而是先统一审视口径，识别哪些视觉语义：
> - 已有 token 可复用
> - 当前缺失，需要新增
> - 命名冲突，需要合并

---

## 1. 审计目标

本次盘点只回答以下问题：

1. 高频组件和页面最常使用哪些视觉语义
2. 这些语义是否已有稳定 token
3. 如果没有，是缺失、重复，还是命名混乱

本模板不直接决定最终实现，只用于形成统一输入。

---

## 4. 建议分组方式

盘点结果建议至少按以下 4 类汇总：

### A. 可直接复用现有 token

适用场景：

- 已有 token 能表达语义
- 只是当前组件没有正确使用

处理建议：

- 优先替换使用方式
- 不急于新增 token

### B. 真实缺失的 token

适用场景：

- 高频语义不存在对应 token
- 多处出现相同或高度相似的视觉值

处理建议：

- 列入首批新增候选
- 优先命名成可复用的语义 token

### C. 重复或冲突 token

适用场景：

- 同一语义存在多套命名
- 不同 token 实际表达同类视觉含义

处理建议：

- 标记主 token
- 其余进入合并 / 退场候选

### D. 暂不处理项

适用场景：

- 低频单点使用
- 业务特殊语义
- 暂时看不出复用价值

处理建议：

- 先记录
- 不进入第一批治理

## 6. 审计结论模板

可直接使用以下格式输出盘点结论：

```md
本轮 token 盘点表明，ArtDeco 当前的主要问题不是 token 完全缺失，而是：

1. 一部分高频视觉语义尚未沉淀为稳定 token
2. 一部分现有 token 没有被正确复用
3. 一部分 token 命名存在重复或冲突

建议第一批治理优先处理：
- 高频颜色与状态语义
- 高复用容器类组件中的边框 / 阴影 / 间距表达
- 会影响页面骨架一致性的 token 缺口
```

---

## 7. 一句话总结

Token 缺口盘点的核心不是“先加更多 token”，而是先确认哪些视觉语义值得进入平台层，哪些问题其实只是现有 token 没有被正确使用。
