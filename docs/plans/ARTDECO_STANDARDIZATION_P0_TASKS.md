# ArtDeco Standardization P0 Tasks

> **说明**
> 本文是 [`ARTDECO_STANDARDIZATION_PLAN.md`](/opt/claude/mystocks_spec/docs/plans/ARTDECO_STANDARDIZATION_PLAN.md) 的 P0 可执行任务清单。
> 它聚焦“标准冻结前的最小收口”，目标不是完成整套 ArtDeco 平台化，而是为后续 P1 / P2 建立稳定前提。

---

## 1. P0 目标

P0 要解决的不是“把 ArtDeco 做完”，而是先回答四个问题：

1. `artdeco-tokens.scss` 当前缺什么
2. Base / Domain 组件边界怎么定
3. 首批应该治理哪些高频组件
4. 后续收敛与脚手架要建立在什么稳定基础上

P0 完成后，应形成：

- 一份 token 缺口清单
- 一份 Base / Domain 组件结构规则
- 一份首批高频组件治理名单
- 一份后续 P1 / P2 的样板基线

---

## 2. P0 工作包

### 工作包 A：盘点 `artdeco-tokens.scss` 缺口

目标：

- 找出当前 token 层没有覆盖、但在高频页面和组件中反复出现的视觉语义

任务：

- 审视高频页面与高频组件中的颜色、边框、阴影、间距、层级、状态表达
- 识别重复字面量和相近语义但不同命名的 token
- 区分：
  - 已有 token 可复用
  - 缺失 token 需新增
  - 命名冲突需统一

产物：

- `artdeco-tokens` 缺口清单
- 建议新增 / 合并 / 淘汰的 token 目录

验收：

- 能明确指出高频组件最常见的 token 缺口
- 能区分“真缺口”与“已有 token 未正确使用”

### 工作包 B：冻结 Base / Domain 组件边界

目标：

- 把组件平台的基本分层先说清楚，避免后续一边重构一边争论边界

任务：

- 盘点现有 ArtDeco 组件目录
- 识别哪些组件应归入 Base 层
- 识别哪些组件应归入 Domain 层
- 明确：
  - Base 可做什么
  - Base 禁止做什么
  - Domain 可依赖什么
  - Domain 应如何表达业务语义

产物：

- Base / Domain 分层规则草案
- 首批需迁移或重命名的组件清单

验收：

- 新组件出现时，团队能快速判断它属于哪一层
- Base 层不再成为业务逻辑沉积区

### 工作包 C：选定首批高频组件样板

目标：

- 为后续 P1 的视觉收敛和测试加固选择最值得先做的一批组件

任务：

- 盘点组件复用频次、影响面和页面分布
- 优先选择：
  - 高复用
  - 易回归
  - 样式差异明显
  - 能代表平台层规则的组件

建议候选类型：

- 按钮类
- 卡片类
- 输入类
- 状态类
- 面包屑 / 加载态 / 通用容器类

产物：

- 首批高频组件样板名单
- 每个组件的治理目标摘要

验收：

- 样板名单具备平台代表性
- 不把边缘组件误当主样板

### 工作包 D：建立 P1 / P2 的执行前提

目标：

- 让后续“视觉收敛”“测试加固”“CLI 固化”建立在明确前提上

任务：

- 将 token 缺口、组件分层、样板清单汇总成统一入口
- 明确 P1 的治理顺序
- 明确 P2 的 CLI 不能在 P0 阶段提前启动

产物：

- P1 优先级表
- P2 前置条件说明

验收：

- 后续推进不需要重新争论基本边界
- CLI 路线不会先于结构稳定而启动

---

## 3. P0 建议输出文档

建议 P0 至少输出以下 3 类文档：

1. Token 缺口盘点
2. Base / Domain 组件结构规则
3. 高频组件样板清单

这 3 类输出比直接改代码更重要，因为它们决定后续 P1 / P2 是否会反复返工。

---

## 4. P0 不做什么

P0 阶段明确不做以下事情：

- 不直接开发 ArtDeco CLI
- 不大规模迁移整个组件目录
- 不把所有现有组件一次性重构
- 不把视觉收敛扩展成全站 cosmetic 调整
- 不在 token 体系未稳定前冻结页面样板

---

## 5. P0 完成标准

当以下条件同时成立时，可判定 P0 完成：

- token 缺口已形成可执行清单
- Base / Domain 组件边界已形成稳定规则
- 首批高频组件样板已确定
- P1 / P2 的前置条件已被明确写清

这时才适合进入：

- P1：样板组件批次治理
- P2：ArtDeco CLI 平台化固化

---

## 6. 一句话总结

P0 的核心不是“开始大改 UI”，而是先把 ArtDeco 平台化需要依赖的语言、边界和样板冻结下来。
