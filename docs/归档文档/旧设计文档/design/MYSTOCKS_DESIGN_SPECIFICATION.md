## 1. 设计原则

### 1.1 核心理念

**专业、高效、清晰**

- **专业性**: 金融数据可视化,精准的数据展示
- **效率至上**: 减少操作步骤,快速获取信息
- **信息清晰**: 数据层级分明,重点突出

### 1.2 设计目标

1. **降低认知负担**: 遵循金融行业惯例,熟悉的视觉语言
2. **提升数据密度**: 在有限空间内展示最多有效信息
3. **减少视觉干扰**: 简洁界面,突出数据本身
4. **保证可访问性**: 支持色盲模式,高对比度

## 8. 响应式设计

### 8.1 断点策略

| 断点 | 屏幕宽度 | 布局调整 |
|------|---------|----------|
| **xs** | <768px | 单列布局,隐藏侧边栏 |
| **sm** | 768-991px | 双列布局,收起侧边栏 |
| **md** | 992-1199px | 三列布局,展开侧边栏 |
| **lg** | 1200-1919px | 四列布局 |
| **xl** | ≥1920px | 固定最大宽度1200px |

### 8.2 移动端适配

#### 导航栏
- xs: 汉堡菜单 (侧滑抽屉)
- sm+: 顶部横向菜单

#### 统计卡片
- xs: 每行1个 (col-24)
- sm: 每行2个 (col-12)
- md+: 每行4个 (col-6)

#### 表格
- xs: 卡片式布局
- sm+: 标准表格,启用横向滚动

#### 图表
- 自适应容器宽度
- 移动端减少数据点
- 触摸友好的交互

### 9.3 K线图规范

```javascript
// K线图配置
{
  candlestick: {
    itemStyle: {
      color: '#F56C6C',      // 上涨 (阳线)
      color0: '#67C23A',     // 下跌 (阴线)
      borderColor: '#F56C6C',
      borderColor0: '#67C23A'
    }
  },

  // 成交量配色
  bar: {
    itemStyle: {
      color: function(params) {
        return params.data.close >= params.data.open
          ? '#F56C6C'  // 上涨红
          : '#67C23A'  // 下跌绿
      }
    }
  }
}
```

  transition: transform 0.3s;
}
.scale-enter-from, .scale-leave-to {
  transform: scale(0.9);
  opacity: 0;
}

## 11. 辅助功能 (Accessibility)

### 11.1 键盘导航

- Tab键: 焦点顺序按视觉顺序
- Enter/Space: 激活按钮
- Esc: 关闭弹窗
- ↑↓←→: 列表导航

### 11.2 屏幕阅读器

- 所有交互元素提供`aria-label`
- 表格提供`caption`和`th scope`
- 图表提供文本替代方案

### 11.3 对比度要求

| 场景 | 最小对比度 | 说明 |
|------|-----------|------|
| **大文本** | 3:1 | ≥18px 或 ≥14px粗体 |
| **小文本** | 4.5:1 | <18px 常规字体 |
| **图标** | 3:1 | 非装饰性图标 |

### 12.2 命名规范
