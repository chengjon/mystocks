## 🧭 Markdown 边界说明写法

新建或重写 Markdown 文档时，优先保证“这份文档是什么、不能替代什么、应该回到哪里看真相源”在开头就写清楚。

### 推荐写法

1. 在文档开头添加边界说明块。
2. 按文档类型选择受控标题：
   - 导航页 / 索引页：`> **导航说明**:`
   - Agent / 工具 / 执行入口说明：`> **使用说明**:` 或 `> **权威来源声明**:`
   - 历史报告 / 历史方案 / 历史总结：使用对应的 `历史*说明`
3. 直接写出权威来源路径，不要写成“前者 / 后者 / 上述文档”。

### 推荐指向

- 仓库级共享规则、审批门禁、治理口径：`architecture/STANDARDS.md`
- 执行流程、命令、协作约束：根目录 `AGENTS.md`
- Claude / OpenCode 执行入口：根目录 `CLAUDE.md`
- OpenSpec 提案、规格、变更流程：`openspec/AGENTS.md`
- 当前实现事实：当前代码与最近一次实际验证结果

### 禁止写法

- 使用“前者 / 后者 / 上述文档”等相对位置表达来代替明确路径
- 用模糊的“以相对顺序决定谁是权威来源”的表述代替显式文件引用

### 本地自检

```bash
python scripts/compliance/markdown_governance_gate.py --root-dir . --format text
```

该检查会验证：
- Markdown 是否包含受控边界说明
- 是否出现基于“前者 / 后者 / 相对顺序判断”的模糊措辞

### 辅助生成

若只是需要快速补一个合规边界说明，可直接使用辅助脚本：

```bash
python scripts/dev/apply_markdown_boundary_note.py --preset navigation path/to/doc.md
python scripts/dev/apply_markdown_boundary_note.py --preset historical --replace-existing path/to/report.md
python scripts/dev/apply_markdown_boundary_note.py --preset auto docs/reports/status.md
```

当前内置 preset：
- `navigation`
- `authority`
- `usage`
- `supplemental`
- `historical`

补充说明：
- `--preset auto` 会根据路径自动推荐边界说明类型；适合批量修复或不想手动判断 preset 的场景。
- 例如 `docs/reports/`、`.planning/`、`archive/`、`worklogs/` 默认会落到 `historical`，`AGENTS.md`、`CLAUDE.md`、`TASK.md` 默认会落到 `usage`。
