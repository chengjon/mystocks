## 🔄 工作流程

### 1. 每日工作流程

```bash
# 开始工作前
git fetch origin  # 拉取其他AI的最新工作
git log --oneline -5  # 查看最近提交

# 工作中
# ... 编写代码 ...

# 完成一个功能
git add .
git commit -m "feat(aiX): 简短描述

详细说明...

🤖 Generated by [AI名称]"
git push origin [分支名]

# 更新进度文件
echo "更新内容" >> .ai-progress.md
git add .ai-progress.md
git commit -m "chore: 更新进度"
git push
```

### 2. 同步其他AI的工作

```bash
# 需要使用其他AI的代码时
git fetch origin
git merge origin/[其他AI的分支]

# 例如：AI 2 需要使用 AI 1 修复的 indicators
git fetch origin
git merge origin/phase4-test-fixes
```

### 3. 冲突解决

```bash
# 如果合并时出现冲突
git status  # 查看冲突文件
# ... 手动解决冲突 ...
git add .
git commit -m "merge: 解决与AI X的冲突"
git push
```

## 📡 沟通渠道

### Git Commit
- 主要沟通方式
- 每个commit说明做了什么
- 使用commit message模板

### 进度文件
- `.ai-progress.md` 实时更新
- 所有AI都可以看到

### 定期同步
- 每天至少 `git pull` 一次
- 每完成一个功能就 `git push`

---

## ⚠️ 重要规则

### DO (应该做)
- ✅ 每个commit都有清晰的说明
- ✅ 完成一个功能就立即push
- ✅ 每天更新.ai-progress.md
- ✅ 使用分支进行开发
- ✅ 需要时合并其他AI的代码
- ✅ 解决冲突时及时沟通

### DON'T (不应该做)
- ❌ 不要在主分支直接修改（除非是AI 3）
- ❌ 不要覆盖其他AI的代码（先pull再push）
- ❌ 不要长时间不push（会导致冲突）
- ❌ 不要删除其他AI的分支
- ❌ 不要修改.ai-collaboration.md（除非大家都同意）
