# ArtDeco Web 优化执行清单（页面/组件可落地版）

- 日期：2026-03-31
- 状态：2026-04-01 已按当前规范链复核
- 对齐基线：当前 ArtDeco 文档体系与运行时代码
- 关联建议文档：`docs/reports/ARTDECO_WEB_DESIGN_OPTIMIZATION_RECOMMENDATIONS_2026-03-31.md`

---

## 0. 执行原则（必须遵守）

1. 不改风格方向，只做一致性与质量提升。
2. reusable assets、page-level shared fragments、domain tab blocks 严格分层，不跨层实现。
3. 所有视觉值优先走 `artdeco-tokens.scss`。
4. 每次只改一个 Domain，避免并行大改。
5. 每个任务都要有“可回滚最小提交”。
6. 不把 `pageConfig.ts` 当成所有 tabs 的唯一真值。

---

## 1. P0 — 先做“稳定收益最大”的项

## 1.1 Token 合规扫描与清理

- [ ] 扫描 `web/frontend/src` 内硬编码颜色、间距、字号、阴影
- [ ] 替换为 token（颜色/字体/spacing/shadow）
- [ ] 建立“禁止新增硬编码样式”的检查规则（lint 或脚本）

**验收**
- [ ] 新增代码硬编码视觉值 = 0
- [ ] 核心页面 Token 使用率 ≥ 95%

---

## 1.2 Trading 工作台视觉语法统一

### 页面层
- [ ] 统一标题层级（主标题/区块标题/说明文字）
- [ ] 统一卡片边框、阴影、背景透明度和圆角语法
- [ ] 统一按钮主次级样式与状态反馈

### 组件层
- [ ] 统一指标卡（数字字体、单位、涨跌标识位置）
- [ ] 统一表格行高、表头层级、hover 高亮策略

**验收**
- [ ] Trading 页面无明显“混搭样式”
- [ ] A 股语义红涨绿跌一致

---

## 1.3 最小回归检查（P0）

- [ ] 交易中心关键操作路径可用（页面切换、Tab 切换、核心按钮）
- [ ] 数据展示无错位、无文本截断、无颜色语义错误

---

## 2. P1 — 结构治理与可维护性提升

## 2.1 组件治理

- [ ] 基于 `ARTDECO_COMPONENTS_CATALOG` 给组件或页面块补齐治理标签：
  - Token 合规
  - A 股语义合规
  - 桌面端断点适配状态
  - 可访问性状态
  - 页面引用状态
- [ ] 清理重复职责组件（统计卡、状态标签、通用面板）
- [ ] Domain 组件与页面块禁止直接写散装视觉样式

**验收**
- [ ] reusable assets 仅包含稳定通用资产
- [ ] 页面系统共享片段和域内工作台块职责清晰

---

## 2.2 页面承载与目录边界修正

- [ ] 明确哪些页面属于模板化工作台
- [ ] 明确哪些页面属于直接 Tab 容器
- [ ] 明确哪些页面属于功能树驱动总控容器
- [ ] `views/artdeco-pages/components/` 只承载页面系统共享片段
- [ ] `views/artdeco-pages/*-tabs/` 只承载域内工作台块
- [ ] `src/components/artdeco/` 只承载沉淀后的可复用资产

**验收**
- [ ] 无跨层依赖反向引用
- [ ] 页面逻辑和视觉资产职责清晰

---

## 2.3 配置驱动收口（pageConfig）

- [ ] `apiEndpoint / wsChannel` 能配置化的保持配置化
- [ ] 清理页面内散落 endpoint 字面量
- [ ] 评审时明确：`pageConfig.ts` 是页面元数据源，不是所有 tabs 的唯一真值

**验收**
- [ ] 页面不再硬编码后端地址 / 频道

---

## 2.4 工程红线自动化

- [ ] 将目录边界规则加入 CI 检查
- [ ] 增加新样式 `@use` 合规检查
- [ ] 增加硬编码视觉值检查
- [ ] 仅把命名规则作为建议项，不做全量机械改名

**验收**
- [ ] 目录规范违规可在 CI 阶段被拦截

---

## 3. P2 — 体验一致性与质量打磨

## 3.1 四大域页面一致性

覆盖：`Market / Trading / Risk / Strategy`

- [ ] 统一信息层级（摘要 > 操作 > 细节）
- [ ] 统一交互节奏（hover / 切换 / 反馈时长与曲线）
- [ ] 统一图表和表格可读性策略

**验收**
- [ ] 四大域页面“同一产品感”明显增强

---

## 3.2 可访问性与性能微调

- [ ] 对比度校验（特别是金色文本与深色背景）
- [ ] 键盘焦点可见性校验（符合 ArtDeco 风格）
- [ ] 降低高开销视觉效果（过重滤镜 / 阴影）

**验收**
- [ ] 核心页面在桌面端常见分辨率下可读性稳定
- [ ] 交互无明显卡顿和视觉闪烁

---

## 4. 页面级任务拆分（建议先后顺序）

1. `Trading`
2. `Market`
3. `Risk`
4. `Strategy`

1. Token 清理
2. 组件统一
3. 页面承载边界修正
4. 回归测试
