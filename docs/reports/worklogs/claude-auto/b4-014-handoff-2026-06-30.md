## 二、冲突处理经验

### 冲突类型

| 类型 | 文件 | 处理方式 |
|------|------|---------|
| Content（import 合并） | `strategy_mgmt.py` | 保留 main 的 `HTTPException` + 添加 B4.014 的 `text`/`inspect` |
| Content（新增代码） | `backtest_repository.py` | 接受 B4.014 版本（HEAD 侧为空） |
| Modify/delete | `.governance/*`、`test_*.py` | 接受 main 删除（方案 F 排除项） |
| 架构分歧 | `openstock_client.py` | 跳过依赖此文件的 commits |

### 关键教训

1. **Cherry-pick 前先检查文件存在性**：M1k 冲突的根本原因是 `openstock_client.py` 在 main 上已删除，但 cherry-pick 时才发现
2. **提取式 cherry-pick**：M1 的 `Lock→RLock` 修复独立于模块重命名，可单独提取 `factory.py` 变更
3. **Governance 文件系统性问题**：每个 B4.014 commit 都修改了 `.governance/` 文件，与 main 的 DI 重构 state 冲突。统一用 `git rm` 处理
