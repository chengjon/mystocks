### A. Full-Path Conflicts (26 groups)

**Action**: Each domain needs P3-A decision record (P3-A5/A6/A7 already in plan).

### B. Prefix-Resolved Non-Conflicts (55 groups)

These share `method + local_path` but resolve to **different full URLs** after prefix expansion. They are not runtime collisions — they are separate endpoints that happen to use the same local decorator path.

Examples:
- `GET /list` → `/api/announcement/list` (package) vs `/api/v1/announcement/list` (VERSION_MAPPING alias)
- `GET /strategies` → `/api/strategy-mgmt/strategies` vs `/api/v1/strategy/strategies`
- `GET /health` in 22 modules → resolves to `/api/{domain}/health`, `/health`, `/v31/health`, etc.

**Action**: No immediate fix needed. May be candidates for semantic alignment (P3-A5 health taxonomy).

### C. Domain Aliases (to classify after P3-A decisions)

Routes that are intentionally duplicated for backward compatibility:

| Alias pair | Mechanism | Status |

| `/api/announcement/*` vs `/api/v1/announcement/*` | VERSION_MAPPING + router_registry dual registration | Unconfirmed consumers for v1; baseline shows `/api/announcement/*` is canonical |

| `/api/multi-source/*` | Flat + package both registered | Full-path conflict (in Category A) |
