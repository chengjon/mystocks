二、关键设计说明（让记录更实用）
1. 编号规则（三级定位法）
格式：页面编号-模块编号-元素编号（如P1-M2-E3）
P（Page）：页面序号（按业务流程排序，如登录页 P1、首页 P2、订单页 P3）
M（Module）：模块序号（页面内的独立功能块，如登录表单 M2、导航栏 M1）
E（Element）：元素序号（模块内的具体元素，如按钮、输入框、图表）
优势：通过编号即可定位元素位置，方便跨表格检索（如搜索 “P2-M1” 快速找到首页第一个模块的所有元素）。
2. 核心字段补充说明
元素类型：明确组件类型（如 Input/Button/Table/Chart/Modal 等），方便关联 UI 组件库（如 Element UI、Ant Design）的参数规范。
参数详情：记录视觉参数（尺寸、颜色、字体）和功能参数（校验规则、数据绑定字段、事件触发条件），AI 生成的参数错误可直接在此标注。
数据来源 / 上下文：
静态元素（如标题）：标注 “固定文本”；
动态元素（如用户昵称）：标注接口地址（/api/user/info）或数据库表（user_table.username）；
交互上下文：如 “点击后打开 Modal（M4）”“数据变化时联动图表（E6）更新”。
依赖项：区分前端依赖（如组件库、工具函数、状态管理模块）和后端依赖（如 API 接口、数据库服务），方便定位问题（如元素异常时先排查依赖是否正常）。
三、进阶优化建议（提升效率）
1. 增加 “层级关系图”（前置目录）

   - 登录表单模块（M2）：账号输入框（E1）、密码输入框（E2）、记住密码复选框（E3）

   - 数据卡片模块（M1）：订单数（E1）、用户数（E2）、销售额（E3）
   - 图表模块（M2）：趋势图（E1）、饼图（E2）

2. 按 “问题类型” 标注备注
AI 生成的常见问题（如参数错误、交互缺失、样式不一致）用标签区分，方便筛选修改：
[参数错误]：如 “maxlength 应为 20，AI 生成 16”
[交互缺失]：如 “未添加点击事件，需关联 handleSubmit ()”
[样式不符]：如 “颜色应为 #1677ff，AI 生成 #000000”
[依赖遗漏]：如 “未关联后端 /api/user 接口”
3. 结合工具提升效率
AI 辅助生成初始表格：用 GPT 等工具先批量识别页面元素（如上传截图或代码，让 AI 提取元素信息），再手动修正。
使用 Markdown 工具的 “表格筛选” 功能：如 Typora、VS Code 的 Markdown 插件，可按 “页面”“模块”“问题标签” 筛选元素，快速定位需修改项。
版本控制：在文件名中加入版本号（如web-elements-v1.2.md），每次修改后记录变更日志，方便回溯 AI 生成的历史版本。
